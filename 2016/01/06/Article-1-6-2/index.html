GitHub + Hexo 搭建免费博客【MAC版】

教程说
  • 网站

阅度前请注意

框中除【标明】的信息外其他均为命令行代码

该教程为天朝资源整理所得，有问题请联系我：QQ:763303669

环境准备

安装Node

到Node.js(EN)官网下载相应平台的最新版本(EN)，下载最新的版本的 .pkg 文件，一路安装即可。

【坑One】：找到正确的下载链接，正确安装Node.js是关键。
          稳定网速下，官网都是秒开，坑网址都是慢得跟坨屎似得。
          ！！！曾深受其害！！！

打开控制台，键入“npm”，首行出现如下代码：

Usage: npm <command>

安装成功

配置GitHub

当然，配置GitHub的前提是需要安装Git的，Mac系统的Xcode就自带了Git，做iOS开发的同学就不用安装了。

回到GitHub

  1. 注册一个『GitHub』帐号，已有的默认默认请忽略。
  2. 建立与你用户名对应的仓库，仓库名必须为『your_user_name.github.io』，例：【tryingx.github.io】。
  3. 添加SSH公钥到『Account settings -> SSH Keys -> Add SSH Key』

前两步忽略，只说第三步，添加SSH-Key。
【首先】设置你的用户名密码：

git config --global user.email "你的GitHub邮箱"
git config --global user.name "你的GitHub用户名"

生成密钥：

ssh-keygen -t rsa -C "你的邮箱"

上述命令会在【~/.ssh】目录下生成两个文件：

其中id_rsa是私钥 id_rsa.pub是公钥

然后，执行下面的命令，将生成的key添加

ssh-add id_rsa

然后将id_rsa.pub里面的内容复制下来，在github上的settings里面找到add keys，将其粘贴到key即可，title随便填

这个时候可以在控制台上测试一下

最后一步：用文本编辑器打开ssh.pub文件，拷贝其中的内容，到GitHub的『Setting -> SSH keys』, 将其添加到Add SSH Key。

【坑Two】：忘记执行ssh-add id_rsa 命令，会一直验证不成功。

Hexo的建站与配置

建立站点文件夹

Hexo建站也非常的简单，执行下列命令， 初始化包含站点文件的文件夹。

hexo init <folder>
cd <folder> 
npm install

你可以在User文件夹下查看到生成的文件夹，主要目录如下：

.
├── _config.yml  #站点配置文件
├── package.json  #应用程序的信息
├── scaffolds  #模板文件夹，博文是根据模板生成的
├── source  #资源文件夹
|   ├── _posts  #生成的博客
└── themes  #下载的主题

现在我们执行下列命令来发布的本地服务器。

hexo g #生成博客静态文件
hexo s #启动本地服务器

在浏览器中打开http://localhost:4000 查看默认效果，是不是炫酷的博客已经出现了！不过这还远不能满足我们，我们需要更个性化的定制。

设置Hexo全局配置

打开站点配置文件_config.yml，只需要修改部分配置，大部分保持默认即可。

# Hexo Configuration 
## Docs: http://hexo.io/docs/configuration.html
## Source: https://github.com/hexojs/hexo/

# Site
title:  #博客标题
subtitle: #博客副标题
description: #博客的描述
author: #作者
language: #网站使用的语言，中文一般使用zh-CN
timezone: #时区

# URL
## If your site is put in a subdirectory, set url as 'http://yoursite.com/child' and root as '/child/'
url: #网站链接，这里我填写的是github的个人页，可以不填
root: /
permalink: :year/:month/:day/:title/
permalink_defaults:

# Extensions
## Plugins: http://hexo.io/plugins/
## Themes: http://hexo.io/themes/
theme: next #主题

使用自己下载的主题时，网站语言可能明明设置zh-CN，但是博客还是没有显示中文简体，这是由于该主题的languages文件中不包含zh-CN，找到该主题下的languages文件夹，查看所包含的中文简体语言文件，比如我现在使用主题的中文就是用的zh-Hans。

写博文

新建博文

由于Hexo使用Markdown，你可以新建.md文件，放在_posts文件夹下，也可以使用以下命令来新建博文。

hexo new "title"

在这里推荐一个Mac的Markdown编辑器Mou，支持双屏浏览，实时保存，不需要手动Ctrl+C保存，否则会有卡顿现象。

Front-matter

打开新建的.md文件，初始内容如下：

---
title: #标题，默认为文件名相同，实际显示以此为准
date: #创建日期
tags: #所属标签   
---

现在开始使用Markdown写你的博文。

该区域用来定义当前文件的变量，文档说明可以使用如下变量参数：

layout: #布局    
title: #标题 
date: #建立日期
updated: #更新日期
comments: #开启文章的评论功能，默认开启
tags: #标签
categories: #分类    
permalink: #覆盖文章网址

多标签注意语法格式如下：

tags:
    - 标签1
    - 标签2
    - 标签3
    - ...

你可以添加你所需要的变量，如果觉得每次生成后更改很麻烦，可以在模板文件夹scaffolds下找到post.md文件，编辑它的Front-matter为你想要的变量参数，默认新建博文使用的post模板。

创建制定模板

当然如果你想使用其他模板，你可以使用下列命令来创建指定模板的博文：

hexo new scaffold "title"

请确保使用的模板scaffold在scaffolds存在。

文章摘要

这个功能很实用，可以让你在首页文章目录不必显示全部的文章内容，只显示指定摘要。

以上是摘要
<!--more-->
以下是全文

注意：所有文件配置：后面，都必须留一个空格，都必须留一个空格，都必须留一个空格，重要的事情说三遍。

主题配置

全局变量配置

同学可以去Hexo主题库下载自己喜欢的主题，我目前使用的主题为NexT，基于本主题介绍主题配置，基本上大同小异。
打开相应主题文件夹下的主题配置文件_config.yml：

# when running hexo in a subdirectory (e.g. domain.tld/blog), remove leading slashes ( "/archives" -> "archives" )
menu: #菜单配置，页头需要显示哪些栏目
home: / #首页
archives: /archives #归档（文章列表）
categories: /categories #分类（需手动创建）
tags: /tags #标签 （需手动创建）
about: /about #关于（需手动创建）
commonweal: /404.html #丢失页面（需手动创建）

# Place your favicon.ico to /source directory.
favicon: /images/favicon.ico #网页图标

# Icon fonts
# Place your font into next/source/fonts, specify directory-name and font-name here
# Avialable: default | linecons | fifty-shades | feather
icon_font: default #字体
#icon_font: fifty-shades
#icon_font: feather
#icon_font: linecons

# Schemes
scheme: Mist #该主题下的子主题

# Sidebar, available value:
#  - post    expand on posts automatically. Default.
#  - always  expand for all pages automatically
#  - hide    expand only when click on the sidebar toggle icon.
sidebar: post #侧边栏展现方式
#sidebar: always
#sidebar: hide

# head image
avatar: http://7xl2f1.com1.z0.glb.clouddn.com/head.jpg #侧边栏头像

menu下需要手动创建的栏目，可以通过执行以下命令：

hexo new page "page"

如果不需要启用Mist主题，在前面使用#注销即可。

个人链接与友情链接

如果需要在侧边栏显示个人链接和友情链接，可以在站点配置文件中增加如下字段：

#个人链接
social: 
GitHub:

# title, chinese available
links_title: 友情链接
# links
links:
link1:

link3:

腾讯公益404页面

使用腾讯公益404页面来替换找不到指定页面的提示，寻找丢失儿童，关注公益事业，效果点击这里查看。
新建一个404.html，放到主题的 source 目录下，内容如下：

<!DOCTYPE HTML>
<html>
<head>
    <meta http-equiv="content-type" content="text/html;charset=utf-8;"/>
    <meta http-equiv="X-UA-Compatible" content="IE=edge,chrome=1" />
    <meta name="robots" content="all" />
    <meta name="robots" content="index,follow"/>
</head>
<body>

<script type="text/javascript" src="http://www.qq.com/404/search_children.js" charset="utf-8" homePageUrl="your site url " homePageName="回到我的主页"></script>

</body>
</html>

第三方服务

评论与分享

这里推荐使用多说，注册账号后，在首页选择“我要安装”，创建站点，填写信息。
打开站点配置文件（不是主题配置文件），增加如下配置：

 #Duoshuo
duoshuo_shortname: name #唯一标示
duoshuo_share: true #是否开启分享
duoshuo_hotartical: true #是否开启热评

其中表单中的多说域名就是需要填写的duoshuo_shortname，这样就可以通过多说来进行分享，回复与热评了。

站点统计

国内推荐使用百度统计，注册百度统计，添加博客地址，然后进入查看代码界面：

打开站点配置文件（不是主题配置文件），增加如下配置：

#Baidu
    baidu_analytics: code

code填写图中红色区域你自己的代码，这样你就可以通过百度统计来查看对于自己博客的各项站点分析了。
关于NexT主题以及更多第三方服务请参考NexT官方文档。

博客图片存储

由于GitHub Pages每个站用户存储空间只有300M，虽然可能图片不多，但是还是推荐使用图床。

根据其他博主的推荐，使用了七牛，注册用户认证之后10G存储空间和每月10G的流量用来做博客图床还是绰绰有余的，同时也提升访问速度。
同学们可以将图片上传到个人存储空间上，由于七牛没有目录的概念，建议图片命名以对应博客的路径命名，便于区别，然后通过链接来访问图片：

<img src="url" class="img-center">

或者

![Alt text](/path/to/img.jpg)

如何部署到GitHub？

站点配置

部署到GitHub上其实也非常简单，打开站点配置文件_config.yml：

# Deployment
## Docs: http://hexo.io/docs/deployment.html
deploy:
    type: github #部署工具的类型
    repository: git@github.com:username/username.github.io.git #SSH链接
    branch: master #分支

创建GitHub Pages

我们托管静态博客，使用的就是GitHub Pages的功能，首先我们需要创建一个个人主页，点击右上角的加号，选择new repositories，进入创建页面

这样我们以后可以通过username.github.io来访问你的主页，点击Create repository创建成功，然后点击侧边栏的setting，进入设置页面：

点击设置成主页，跳转后无需填写信息直接点击Continue to Layouts为主页选择一个主题：

点击publish pages就创建个人主页成功了，这也是我们托管自己博客的仓库，你可以通过username.github.io来先浏览刚生成的个人主页。

部署到GitHub

打开站点配置文件_config.yml：

# Deployment
## Docs: http://hexo.io/docs/deployment.html
deploy:
    type: github #部署工具的类型
    repository: git@github.com:username/username.github.io.git #SSH链接
    branch: master #分支

在repository处填入：git@github.com:username/username.github.io.git，保存后打开控制台键入以下命令：

cd blog #进入博客站点文件夹
hexo g #生成静态网页
hexo d #部署

如果出现以下信息：

INFO  Deploying: git
INFO  Clearing .deploy folder...
INFO  Copying files from public folder...
........
INFO  Deploy done: git

恭喜你，博客已经部署成功，快通过username.github.io访问你的博客吧！

域名绑定

命令行 CD 到项目的根目录（root）下:

touch CNAME

回车：

vim CNAME

回车,输入你要绑定的域名：

www.tryingx.com

ESC -> :wq

保存退出，并同步到GitHub项目下。
解析速度很快，大功告成。大家快快访问：www.tryingx.com吧。
